如何搭建https支持的个人博客

发表于 2019-04-30 | 分类于 Hexo | 评论数： | 阅读次数：

有利于提高生产力的知识应该被自由且有效地分享

真实的情感或者有用经验，值得我们通过文字和代码的方式传达给有需要的人，以减少信息熵增对我们个体不利的影响。如果你也这么觉得，何不一起来搭建一个属于自己博客（向网络世界展示自己的一个窗口），向世界分享你的见解？又或者只是单纯的把这段经历当作是学会使用某些工具的试验行为，丰富业余的生活。

本文旨在介绍使用 Github Pages 服务，搭建一个基于 Hexo+NexT 的个人博客。

概述

为什么是Github

Github是全球最大的软件源代码托管（同性交友）网站，同时也面向学生提供非常实用的Student Developer Pack 。学生用户通过.edu 邮箱注册即可。
拥有Student Developer Pack的学生可在Namecheap上免费申请到一个以 .me 结尾的个人域名，免费使用有效期一年。

Github还支持 Pages 服务，简单来说就是Github可以把用户托管的软件代码转换成一个链接为用户名.github.io的静态网站。
2018年的国际劳动节，Github宣布在Pages服务中，如果用户使用自定义域名，也将会直接得到 Github提供的HTTPS 支持。如此一来，用户无需绕道使用CloudFlare等服务商就可以获得Github提供的HTTPS支持。这种经济实用的方式非常适用于个人博客开发者和学生党。

自定义域名

如前所述，在校高校生可以使用Student Dev Pack在Namecheap中申请到为期一年的免费域名，并且Namecheap也提供免费的DNS解析服务和WhoisGuard服务。在Namecheap中申请成功后，参照以下操作设置指向Github的DNS。

在Dashboard中打开Advanced DNS设置列表

设置DNS

参照Github Pages帮助文档，在HOST RECORDS设置中填写三条records。

  1. 点击 ADD NEW RECORD, Type为 A Record类型，Host 为@, Value以下四条中的任意一条，点击绿色的✔：
    • 185.199.108.153
    • 185.199.109.153
    • 185.199.110.153
    • 185.199.111.153
  2. 重复以上，添加第二条185.199.1XX.153 。
  3. 第三条record, Type为 CNAME Record，Host 为 www, Value 为已经建好的Github 博客默认地址 userID.github.io.，特别注意的是地址最后需要加一个点，就像这样.io.。

至此，自定义域名的DNS已设置完毕，由于DNS解析需要一定的时间后才会生效，现在只要安静的看个番，恰点茶就可以了。
